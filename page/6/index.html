  1. 一些有用的工具

    1. node-dev

      • 当文件更新时自动重启的 node
      • 避免每次改完代码都要重新运行的麻烦
      • 不宜在生产环境使用

    2. ts-node

      • 让 node 支持直接运行 TypeScript 代码
      • 不宜在生产环境使用

    3. ts-node-dev

      • 这个工具结合了上面两个工具
      • 可以用 TypeScript 开发 Node.js 程序，且会 自动重启
      • 不宜在生产环境使用，但非常适合用来学习

    4. WebStorm 自动格式化

      • Reformat Code 设置为 Ctrl + L 或者其它键位,默认为 alt+command+L
      • 可以快速帮你格式化 TS/JS 代码

    5. WebStorm 自动写代码

      • Complete Current Statement 设置为 Ctrl+Enter,完全完成当前语句
      • Typescript 会自动匹配最精确,按 alt+enter 之后再按 enter 可以实现补全代码，自动声明变量
      • Show Context Actions 设为 Alt+Enter,
      • 把冲突的键位改成其它快捷键
      • 当然你可以设为其它键位，这只是我的习惯

    6. VSCode 配置

      • 开启自动保存，开启保存时自动格式化（有些人不喜欢）
      • 当然也可以 Shift+alt+F 手动格式化
      • Quick Fix 默认快捷键 Ctrl+句号（功能稍弱）

    7. WebStorm Git 配置

      • 把 WebStorm 的 git 路径设置为 cmder 里的
      • 例子:/usr/local/bin/git
      • 好处是统一管理统一配置

    8. VSCode Git 配置

      • 在 settings.json 里添加
      • 目的是让 VSCode 使用 cmder 里的 git 和 bash
      • 如果没有 cmder 就自行修改为其他路径

    9. curl

      • GET 请求: curl -v url
      • POST 请求: curl -v -d “name=ories” url
      • 设置请求头: -H ‘Content-Type: application/json’
      • 设置动词: -X PUT
      • JSON 请求: curl -d ‘{“name”:”bob”}’ -H ‘Content-Type:application/json’ url
      • 后面会用到 curl 来构造请求

  2. 创建项目

    1. 步骤

      • yarn init -y
      • 新建 index.ts
      • 使用命令行或者 WebStorm 启动
      • yarn add –dev @types/node 安装 node 声明文件
      • 引入 http 模块(WebStorm 自动导入)
      • 用 http 创建 server(WebStorm 自动命名)
      • 监听 server 的 request 事件(可以简写)
      • server.listen(8888)开始监听 8888 端口
      • 使用 curl -v http://localhost:8888 发请求
      • 实现最简单的 node.js 服务器
      • 然后，ts-node-dev index.ts
      • 然后在控制台 curl http://localhost:8888, 看到反应

  3. request 对象

    1. server 是个什么东西

      1. http.Server 类的实例
      2. 根据文档知道 http.createServer 的返回值的类型
      3. 所以 server 是 http.Server 的实例
      4. 因此 server 拥有几个事件和方法
      5. 其中也就 request 事件和 listen()方法目前用的上

    2. 继承 net.Server 类

      1. 根据文档知道 http.Server 继承了 net.Server
      2. 因此 server 有用了几个事件和方法
      3. 其中也就 error 事件和 address()方法目前能用上,address 看监听的对象是什么

    3. 看文档的方法，看有哪些事件，属性，方法

      • 有用的事件, request 有人请求了怎么办, listen 开始监听
      • curl + -v 可以打印出请求的内容
      • 处理 post 请求无法发现，请求体

  4. 如何得到请求消息体

  5. 用 Node.js 获取请求内容

    • request.method 获取请求动词
    • request.url 获取请求路径（含查询参数）
    • request.header 获取请求头
    • get 请求一般没有消息体/请求体

  6. post 请求

    • curl -v -d “name=ories” http://localhost:8888

    • request.on(‘data’,fn)获取消息体

    • request.on(‘end’,fn)拼接消息体

  7. (request, response)是啥

    1. 找类

      • 根据文档， request 是 http.IncomingMessage 的实例
      • 根据文档, response 是 http.ServerResponse 的实例

    2. Request

      • 拥有 headers,method,url 等属性
      • 从 stream.Readable 类继承了 data/end/error 事件
      • 为什么不能直接拿到请求的消息体，跟 TCP 有关

    3. Response

      • 拥有 getHeader/setHeader/end/write 等方法

      • 拥有 statusCode 属性，可读可写

      • 继承了 Stream,目前用不上
