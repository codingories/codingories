React全解之03Hooks原理解析

Posted on 2020-03-20 Edited on 2020-10-02 In 前端框架 Views: Valine:

1. 案例一.点击按钮+1

  • 同样的 n 在不同的次数执行，得出的结果是不一样的
  • 首次渲染 render<App/>组件, App 组件会调用 App 函数，执行之后得到虚拟 DIV 创建，创建真实 DIV
  • 用户点击 button 调用 setN(N+1) 再次 render<App />
  • 调用 App(),得到虚拟 DIV,DOM Diff 更新真 DIV,每次调用 App(),都会运行 State(0)
  • 每次调用 App(),都会运行 useState(0),每次 useState 的 n 不一样

2. 问自己几个问题

  • 引出问题: 执行 setN 的时候会发生什么？n 会变吗？App()会重新执行吗？
    • 执行 setN 表示要把 n 变了，这个时候 n 是不变的，即 setN 并不会改变 n,App 会重新执行
  • 如果 App()会重新执行，那么 useState(0)的时候，n 每次的值会不同吗?
    • 确实 n 的值是不同的
  • 通过 console.log 得到答案
  1. 分析
  • setN
    • setN 一定会修改数据 x, 将 n+1 存入 x
    • setN 一定会触发<App/ > 重新渲染
  • useState
    • useState 肯定会从 x 读取 n 的最新值
  • x
    • x,每个组件有自己的数据 x，我们将其命名为 state

3. 首次尝试实现 useState

  • 这样写之后还是不行，因为每次 render,n 都是初始值，解决方法是把 state 放到外面, 只要声明在 myUseState 外面即可
  • 进行这样改写就可以了。

4. 如果一个组件用了多个 useState 怎么办？

  • 有两种改进思路
  1. 把_state 做成一个对象
  1. 把_state 做成数组
  • 如何让多个 useState 不冲突

5. _state 数组方案的缺点

  • useState 调用顺序
    • 若第一次渲染时 n 是第一个,m 是第二个,k 是第三个
    • 则第二次渲染时必须保证顺序完全一致
    • 所以 React 不允许出现如下代码, react hooks 不能写在条件循环
  • 题外话:Vue3 克服了这个问题。所以，尤雨溪说 Vue3 借鉴了 React 的思想，但是比 React 做的更好。

6. 现在代码还有一个问题

  • App 用了_state 和 index, 那其他组件用什么？

    • 解决办法:给每个组件创建一个_state 和 index

  • 又有问题:放在全局作用域里重命名了怎么办？

    • 解决办法:放在组件对应的虚拟节点对象上

  • 上图只画了 App 组件的更新过程，两个 ChildA 组件也有同样的过程

7. 总结

  • 每个函数组件对应一个 React 节点
  • 每个节点保存着 state 和 index
  • useState 会读取state[index]
  • index 由 useState 出现的顺序决定
  • setState 会修改 state,并触发更新
  • 注意: 本篇文章对 React 的实现做了简化，React 节点应该是 FiberNode,_state 的真实名称为 memorizedState, index 的实现则用到了链表

8. useState 错误理解案例

  • 其实和 for 循环 setTimeOut 0, 函数打印出 6 个 6 一样

9. 如何有贯穿始终的状态

  • Vue3 就是每次都是一次 n
  • React 底层思想非常倾向函数式所以不希望 n 多次变化
  • 解决办法:
    • 全局变量, window.xxx, 太 low 了
    • useRef 例子,
      • https://codesandbox.io/s/flamboyant-elion-jvtl2
  • 手动造出来让 useRef 更新 dom

  • Vue3 主要抄了 React 两个地方

    • hooks 的思想
    • 不使用 useState,而是用 useRef，再你自己对 useRef 的 current 的值做变更的时候，自动的去 update

  • 通过 useContext 实现贯穿始终,useRef 只是贯穿一个组件的前中后,useContext 是贯穿所有组件,大多数情况还是用 useRef,除非是非常重的方法用 useContext

  • 强制更新的例子,https://codesandbox.io/s/wizardly-grothendieck-ek0i5

10. 总结

  • 每次重新渲染，组件函数就会执行
  • 对应的所有 state 就会出现[分身]
  • 如果不希望出现分身，可以用 useRef/useContext 等解决

React全解之01Class组件详解

Posted on 2020-03-11 Edited on 2020-10-28 In 前端框架 Views: Valine:

零. 元素与组件

  • const div = React.createElement('div',...),这是一个 React 元素(D 小写)
  • const Div = ()=>React.createElement('div',...),这是一个 React 组件(D 大写)
  • 就目前而言，一个返回 React 元素的函数就是组件
  • 在 Vue 里，一个构造选项就可以表示一个组件

一. 需要注意写 React 标签的时候，一定要注意不是写 HTML 标签

  • <div />会被翻译为 React.createElement(‘div’)
  • <Welcome />会被翻译为 React.createElement(‘Welcome’)
  • 可以用 bebel online 直接翻译
  • React.createElement 的逻辑
    • 如果传入一个字符串’div’,则会创建一个 div
    • 如果传入一个函数，则会调用该函数，获取其返回值
    • 如果传入一个类，则在类前面加个 new(这回导致执行 constructor),获取一个组件对象，然后调用对象的 render 方法，获取其返回值

三. 小试牛刀，通过类组件和函数组件，分别实现+1 功能

四. React 组件如何使用 state 内部数据

  • 这里的 State 和 Vue 里面 Data 一个意思

五. React 和 Vue 的编程模型对比

  • Vue 的编程模型
    • 一个对象，对应一个虚拟 DOM,当对象的属性改变时,把属性相关的 DOM 节点全部更新。注意:Vue 为了其他考量，也引入了虚拟 DOM 和 DOM diff,下图只是简化
  • React 的编程模型
    • 一个对象，对应一个虚拟 DOM。另一个对象，对应另一个虚拟 DOM。对比两个虚拟 DOM,找不同(DOM diff),最后局部更新 DOM

六. 复杂的 state

  • 类组件里有 n 和 m
  • 函数组件有 m,n

七. 总结

  • 类组件里的 setState 会自动合并第一层属性
  • 但是并不会合并第二层属性
  • 使用 Object.assign 或者…操作符合并第二层属性
  • 函数组件的 setX 则完全不会合并
  • 要合并必须自己用…合并

八. React事件绑定各种写法

  • 类组件的事件绑定

  • 当用户点击<button onClick={this.addN} />React会做什么？

    • 会运行button.onClick.call(null,event), 浏览器this===null会变成window，所以第二种写法this会变成window

  • 函数组件注意事项

    • 函数组件跟类组件相似的地方，也是通过setX(新值)来更新UI
    • 跟类组件不同的地方，没有this,一律使用参数和变量

1. 什么是 Redux

  • Redux 是 JavaScript 状态容器，提供可预测化的状态管理。
  • https://www.redux.org.cn/

2. 三个概念

  • action 用来描述做什么事情,描述晚了之后到 reducer,
  • reducer 就是描述完了，接收描述，并且做一定的数据处理，主要是用来执行 action 的命令，
  • state 是用来储存数据状态

3. +1 案例

  • 用 redux 实现+1
  • 不用 redux 实现+1

4. 为什么要用 Redux

  • 在 react 中一般都是单页面应用，数据量非常大，声明全局变量不好
  • react 中还要设计兄弟组件，爷爷孙子组件数据共享的问题，Redux 就很方便管理

5. store 里面 subscribe 接口的作用

  • 添加一个变化监听器，每当 dispatch action 的时候就会执行

6. react 搭配 Redux 的例子

7. 在项目中结合,使用 ReactRedux

  • connect 的作用，也是一个组件，把我们要用的组件外面包一层，然后导出，需要两个参数，mapStateToProps,mapDispatchToProps,这样子 counter 就可以通过 connect 获得 state,mapStateToProps 获取 props,mapDispatchToProps 获取 dispatch 的函数。
