Posted on 2020-07-11 Edited on 2020-07-12 In Next.js Views: Valine:

一.Next.js 的背景

  • zeit 团队水平如何？请看这个。后改名为 Vercel
  • 简而言之，一个高中开始变成的，会做平面设计的复旦大学计算机专业毕业生，在微软工作过一年后，加入 zeit 团队
  • 几乎每一个同事都有非常强大的背景
  • Next.js 核心团队四个人平均年龄 20 岁
  • 按 star 数，zeit 是 GitHub 组织的 Top20
  • 全员远程工作

二. Next.js 的定位

  • Node.js 全栈框架
    • CSS-in-JS
    • 页面预渲染 + SSR (服务端渲染)
    • 前后端同构建(代码同时运行在两端)
    • Node.js10.13 以上
    • React yes!
    • TypeScript yes!
  • 弱项
    • 完全没有提供数据库相关功能，可自行搭配 Sequelize 或 TypeOrm
    • 完全没有提供测试相关功能，可自行搭配 Jest 或 Cypress
    • 有一个叫做Blitz.js的框架在这些方向上努力

三. 谁在用 Next.js

  • 国内
    • 腾讯网手机版https://xw.qq.com
    • Teambition
  • 国外
    • Netflix Jobs, Twitch 手机版， Hulu, TikTok 抖音海外版，漫威官网， Material UI 官网
  • 总结
    • 国外用得比较多，国内用得少
    • 那么国内学这个好找工作么？

四. 关于国内学 Next.js 是好找工作

  • 思考
    • 国内一般招聘偏后端的前端，看看纯 Node 后端市场
    • Next.js 切合了 SSR,Serverless 等技术热点
    • 支付宝的 Umi.js 是一个类 Next.js 框架，说明被认可
    • Next.js 背后团队
    • 已经会 express 和 koa，可自行掌握 egg.js
  • 综上
    • 进可往技术热点靠
    • 退可回到 egg.js

五. 使用 Next.js 做博客系统

  • 官方教程
    • 官方入门教程就是做博客系统
  • 需求
    • 我们对博客系统的需求足够了解，不需再讨论
  • 难易程度
    • 非常简单，易于学习

    • 用浏览器打开控制台给出的网址

八. Link 快速导航

  • 官方教程在此
  • 用法
    • 把点击跳转改成
    • 点击跳转
  • 优点
    • 页面不会刷新，用 AJAX 请求新页面内容
    • 不会请求重复的 HTML,CSS,JS
    • 自动在页面插入新内容，删除旧内容
    • 因为省了很多请求和解析过程，所以速度极快
  • 吐槽
    • 借鉴了 Rails Turbolinks，Pjax 等技术
  • 传统导航
  • 快速导航

九. 同构代码

  • 代码运行在两端
    • 在组件里写一句 console.log(‘执行了’)
    • 会发现 Node 控制台会输出这句话
    • 会发现 Chrome 控制台也会输出这句话
  • 注意差异
    • 不是所有的代码都会运行，有些需要用户触发
    • 不是所有的 API 都能用，比如 window 在 Node 里报错

十. 小结

  • 创建项目
    • npm init next-app 项目名
  • 快速导航
    • <Link href=xxx>包住</a>
  • 同构代码
    • 一份代码，两端运行(省了一份)

十一. 全局配置

  • 官方教程
    • 自定义<Head>
    • 全局 css
    • 局部 css

十二. 自定义<Head>

  • 使用 <Head>

    • 更改各个页面 title
    • 更改页面 meta:viewport

  • 全局配置

    • page/_app.js
      • export default function App 是每个页面的根组件
      • 页面切换时 App 不会销毁，App 是里面的组件会销毁
      • 可用 App 保存全局状态
    • 注意
      • 创建_app.js 之后需要重启 yarn dev

十三. 全局的 css

  • 放在_app.js 里
    • import ‘../styles/global.css’
    • 因为切页面时 App 不会销毁
    • 其他地方不能 import global.css
    • 其他地方只能写局部 css
  • 插曲
    • 相对引用比较麻烦，改成 import ‘styles/global.css’，查看文档

十四. 局部 css

  • 官方支持
    • 默认支持 styled-jsx 和 CSS Modules
    • 一般来说，简单需求用前者，复杂需求用后者
  • 个人偏好
    • styled-jsx 不方便分离 CSS 和 JS
    • CSS Modules 用起来太麻烦
    • 更喜欢用 styled-components

十五. 使用 SCSS

  • 安装依赖
    • yarn add sass
  • 直接用
    • 把后缀.css 改为.scss 即可

十六. 静态资源

  • next 官方推荐放在 public/里，但不是很好，因为不支持改 hash 名，导致不会更新
  • 其他类型文件
    • 图片
      • 配置 file-loader
      • 或者直接用 next-images
    • SVG
      • 一部分可以当作图片使用
      • icons/svg 要当作 SVG Symbols 使用
      • 目前没有插件，只能自己配置
    • 更多
      • 自己要 loader,然后配置 next.config.js
      • 或者看有没有人封装成 next 插件
