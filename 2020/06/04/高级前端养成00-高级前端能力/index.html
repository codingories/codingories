一. 高级前端面试痛点举例

  • 简历没亮点
  • 手写 Promise
  • Eventloop 了解吗
  • webpack 原理说一下

二. 高级前端面试重点非常强调主动

  • 主动写了
  • 主动研究了
  • 主动承担了

三. 简历没亮点

  • 什么项目算亮点
  • 管理平台不算亮点，官网不算亮点
  • 用数据说话算亮点
  • 为团队造轮子算亮点
  • 提升性能两倍算亮点
  • 参与开源项目算亮点

四. 哪些工作算亮点

  • 负责开发工作不算亮点，这是本职工作
  • 负责面试算亮点
  • 团地分享算亮点（演讲或文章）
  • 向团队安利新技术算亮点
  • 对成员进行培训算亮点
  • 总结:平时代码写烂点，突然写好点算亮点，造两轮子算亮点，造完轮子分享 ts 使用心得算亮点，面试把把关也是亮点，向团队安利新技术算亮点

五. 如何获得这些亮点。

  • 平时积累轮子，参加开源项目，多做分享。 六. 面试:
  • 常见的题目
  • 手写 Event Hub
  • Promise
  • 手写 bind 和 call
  • 手写深拷贝
  • Ajax 请求(封装出一个 axios)

七. 面试解法

  • 平时写博客，这样至少有思路
  • 面试前刷题，手写一遍是最有效的
  • 工作中大部分只需要知道原理，需要手写
  • 举例 Promise 原理 问题 callback hell 同类型解法 1.node js 2. generator 不够直观 3. es 6 promise 优点（看起来同步)缺点(还不是同步，await 就是语法糖)你会手写 Promise 吗
  • 举例 webpack 原理 问题。模块，打包，以前 gulp,require,grunt,都是任务化，不符合前端思想 几个月就死掉了同类型解法。 commands amd 都是民间的标准 ES module 优点:
    1. 全兼容以上三种格式
    2. 管道思想
    3. 缺点慢
    4. 慢怎么办 分层打包，cdn 全局属性

九. 心得体会系列

  • 常见的题目
  • 你遇到的最复杂的技术难题
  • 如何应对需求变更快
  • 如何与同事相处
  • 如何快速定位 bug
  • 如何控制项目的复杂度
  • 如何优化性能
  • 工程化怎么做
  • git 规范怎么做
  1. 心得体会解法
  • 没想过就不会答，平时就要思考
  • 看看别人怎么答的
  • 举例:如何应对需求变更快
    • (调休）加班-最蠢的答案解决，
    • 假想，产品经理 提的需求，流程约束，需求变更流程->发邮件，为改进提供依据。 记录每一次变更的原因，以及优先级是否这么强烈，当前项目这样，下个项目更好。对产品进行建议，教育，一开始没有想清楚，加快自己的工作效率，重复的业务变成轮子，开发上的其他工具脚手架
