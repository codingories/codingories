数组典型面试题目

两数之和给定一个整数数组（无重复元素）和一个目标值，找出数组中和为目标值的两个数。 按照从小到大的顺序输出结果对可以假设每个输入只对应一种答案例:

思路1：bruteforce

  • 暴力遍历：遍历取一个数，计算它与其它数字之和，遍历全部情况得到想要的结果对
  • 时间复杂度：O(n^2)

    思路1总结

  • 时间复杂度：O(n^2)
  • bruteforce暴力解法
  • 存在无意义的操作
    • 例：nums:{1,4,20,15,8,6,3},target=10.
    • 第一次循环：1+20>10
    • 之后的循环：4+20,没有必要

思路2：排序+两根指针

  • 通过对数组排序与两根指针组合，减少无意义的遍历
  • 两根指针：排序后，一根指针（start指针）指向数组第一个元素（数组中最小元素），另一个指针（end指针）指向数组最后一个元素（数组中最大元素）
  • 核心想法：如果现在两根指针所指元素之和大于目标值，则表明现在两数之和过大，应使end指针指向更小的数，即索引减小（end–），反之则表明现在两数之和过小，应使start指针指向更大的数，即索引增加（start++）

    思路2总结

  • 通过对数组排序与两根指针组合，减少无意义的遍历
  • 使用两个指针（而不是一个）以相同/相反的方向遍历数组一根指针（start指针）指向数组第一个元素（数组中最小元素），另一个指针（end指针）指向数组最后一个元素（数组中最大元素）
  • 时间复杂度分析：排序：O(nlogn),两根指针算法：O(n)
  • 时间复杂度：O(nlogn)+O(n)=O(nlogn)

三数之和

三数之和给定一个包含n个整数的数组（无重复元素）nums和一个目标值target，找出数组中和为目标值的三个数。 可以假设每个输入只对应一种答案 例如,给定数组nums=[-1,0,1,2,-4]，target=0 满足要求的三元组集合为：[-1,0,1]

思路

  • 当然可以用暴力遍历求解，时间复杂度为O(n^3)
  • 看看排序+两根指针算法是否能够求解遍历第一个数字num1，看看另外两数之和是否能满足target–num1，这就转化为两数之和的问题
  • 时间复杂度：
    • 对于不同的n个第一个数：排序+两数之和
    • O(nlogn)+n*O(n) = O(n^2)

K-Sum解法总结

  • 排序
  • 尝试遍历第一个数，将问题转化为k–1Sum
  • 时间复杂度：
    • 2-Sum:O(nlogn)+O(n)=O(nlogn)
    • 3-Sum:O(nlogn)+O(n^2)=O(n^2)
    • 4-Sum:O(nlogn)+O(n^3)=O(n^3)
    • k-Sum:O(nlogn)+O(n^(k-1))=O(n^(k-1))

变量声明与赋值中-LHS和RHS

  • LHS是指Left-handSide，而RHS是指Right-handSide。二者区别就是关于作用域对变量的查询目的是变量赋值还是查询。
  • LHS可理解为变量在赋值操作符(=)的左侧，例如a=1,当前对变量a查找的目的是变量赋值。
  • RHS可以理解为变量在赋值操作符(=)的右侧，例如:int b=a,其中对变量a的查找目的就是查询。

  • 基本类型：将RHS的值直接赋给LHS
  • 引用类型：将RHS对象的地址引用安排给LHS

  • 一度，C和C

  • 三度, 大三度=五个音，小三度=四个音

  • 小三度记忆包含原本的两个音在内一共有四个半音，大三度有五个半音

  • 和弦:各种度的组合,多个三度叠加而成

  • 三和弦

  • X大三和弦=”X为根音网上叠大三度再叠小三度”

  • 大三和弦:大+小,X代表12个半音中任意一个音

  • 小三和弦= 小三度+大三度，Xm,Am,Dm

  • 增三和弦=”大三度”+”大三度”,Xaug,Aaug,Caug

  • 减三和弦 = “小三度”+”小三度”,Xdim

  • CEG=C大调中的1级和弦，为C大三和弦
  • DFA=C大调中的2级和弦，为D小三和弦

  • 七九和弦离调和弦更丰富，更耐听

  • 五度圈就是251的和弦进行

  • 如何运用
    • 找到一个和弦当作临时主和弦
    • 根据五度圈推算出前面要改变的两个和弦
    • 当临时主和弦是小和弦的时候，2级用减三或者半减7，5级用属7或者属7降9
    • 当主和弦是大和弦的时候，二级用小三，小七，半减七，5级用属7或者属9和弦

    • 半减7和弦=减三和弦+大三度，写法为Xm7-5,意思是小七和弦五度音降低半音

    • 也可以直接用五一,省略2
  1. 作曲与编曲的区别
  • 作曲-指把自己想到的前人没有创作过的曲子，用简谱或五线谱等方法记录下来并表达出来。当然现在很多歌手在作曲的时候，会直接即兴哼唱旋律录制下来而比较少用谱子
  • 误区：编曲是作曲的一部分，正解:作曲是把旋律无中生有想出来，说白了就是歌手要长的旋律。而编曲则是依照旋律，添加各种音色，配上和弦，做出一首完整的伴奏。
  • 作曲:compose(组织)，即对素材进行整合，组装，创造性地安排，使用
  • 现代音乐，创作出来的音乐是否能够真实演奏已经无关紧要
  • 编曲：不影响原曲旋律的情况对乐曲的表现形式做扩展
  • 作曲：骨架，编曲：血肉
  • 总结：当今流行音乐，作曲就是要做出人要唱的主旋律，有时候也可以理解为背景音乐类型当中solo乐器的主旋律。而编曲就是实录或者软件当中制作出完整伴奏。

9. DOM diff大概逻辑

  • Tree diff
    • 将新旧两棵树逐层对比，找出哪些节点需要更新
    • 如果节点是组件就看Component diff
    • 如果节点是标签就看Element diff
  • Component diff
    • 如果节点是组件，就先看组件类型
    • 类型不同直接替换（删除旧的）
    • 类型相同则只更新属性
    • 然后深入组件做Tree diff(递归)
  • Element diff
    • 如果节点是原生标签，则看标签名
    • 标签名不同直接替换，相同则只更新属性
    • 然后进入标签后代做Tree diff(递归)

10. Dom diff的缺点

    • 会出现识别错误
    • 举例，砍掉左腿右腿变成左腿
    • 解决方案，命名，给id,绑定key,注意永远不要以数组的index绑定key
